Quartz：原理介绍与学习
java 定时作业 Quartz 第三方框架
Quartz
发布日期:   2020-03-16
阅读次数:  

前言

​ 最近公司业务需要引入定时任务，所以决定采用比较成熟的Quartz框架，在此进行整理学习。

​ Quartz是一个开源的定时调度框架，支持集群部署，可以和Spring进行集成，通过调用Java API来进行使用。

1.Quartz简介

1.1 基本元素与关系

​ Quartz主要元素：

  • Trigger：触发器，用来定义执行指定作业的计划

  • Schedule：任务调度的控制器

  • Job：元数据，指定作业

    关系如下：

1.2 Trigger(触发器)

​ 触发器用于定义任务调度的时间规则，比如每天、没小时，间隔多久等等。

​ 最常用的两种触发器是：SimpleTrigger和CronTrigger,两者均继承了Trigger接口。

  • SimpleTrigger

    ​ SimpleTrigger是一种最基本的触发器，唯一的实现类是SimpleTriggerImpl，指定从一个时间开始，以一定的时间间隔执行任务。

    public class SimpleTriggerImpl extends AbstractTrigger<SimpleTrigger> implements SimpleTrigger, CoreTrigger {}

    SimpleTrigger的实例一般通过TriggerBuilder来实例化，代码如下：

    //创建triggerBuilder,jobName为作业名，jobGroup为作业组名
TriggerBuilder triggerBuilder = TriggerBuilder.newTrigger().withIdentity(jobName, jobGroup);
//设定触发器的调度规则，采用SimpleTrigger
SimpleScheduleBuilder ssb = SimpleScheduleBuilder.simpleSchedule();
//设定trigger执行间隔时间为 jobObject.getIntervalInSeconds() 秒             
ssb = ssb.withIntervalInSeconds(jobObject.getIntervalInSeconds());
//入参
triggerBuilder.withSchedule(ssb);
//指定开始执行时间
triggerBuilder.startAt(startTime);
//创建trigger
Trigger trigger = triggerBuilder.build();

  • CronTrigger

    ​ CronTrigger适用于复杂的时间表达式，即支持cron表达。其唯一的实现类为CronTriggerImpl。

    public class CronTriggerImpl extends AbstractTrigger<CronTrigger> implements CronTrigger, CoreTrigger {}

    ​ CronTrigger是实例化也要通过TriggerBuilder。

    //同上
TriggerBuilder triggerBuilder = TriggerBuilder.newTrigger().withIdentity(jobName, jobGroup);
//通过CronScheduleBuilder设置执行规则,采用CronTrigger，jobObject.getCron()为cron表达式
triggerBuilder.withSchedule(CronScheduleBuilder.cronSchedule(jobObject.getCron()));
//指定开始执行时间
triggerBuilder.startAt(startTime);
//创建trigger
Trigger trigger = triggerBuilder.build();
1.3 Job(元数据)

​ Job是一个接口，其内部只有一个execute()方法：

public interface Job {
    void execute(JobExecutionContext var1) throws JobExecutionException;
}

​ 我们需要实现该接口，编写业务代码，指定定时所要处理的业务。

​ 下面是几个常用的类：

  • JobDetail：任务细节，Quartz在执行Job时，需要创建Job实例，但是不能直接操作Job（接口）类，所以通过JobDetail来获取Job的名称、描述信息等。

  • JobKey：作为JobDetail的标志类，JobDetail在创建时需要指定JobName和GroupName，该类的作用即用来存储这两个字段，作为JobDetail实例的标识ID，作为创建JobDetail的入参。

    //jobObject为自建类
String jobName = jobObject.getJobName();
String jobGroup = jobObject.getJobGroup();
//获取实现了Job接口的类的class（我们要实现Job的那个类）
Class clazz = Class.forName(jobObject.getJobHandler());
//创建jobKey实例
JobKey jobKey = new JobKey(jobName, jobGroup);
//通过jobKey创建JobDetail
JobDetail jobDetail = JobBuilder.newJob(clazz).withIdentity(jobKey).build();
1.4 Schedule（任务调度）

​ 定时框架Quartz的核心概念，目的是将JobDetail和Trigger注册带scheduler中，保证定时任务的执行。

@RootComponent
public class SchedulerManager extends ApplicationListenerBase {

    //Spring注入SchedulerFactoryBean
    @Autowired
    private SchedulerFactoryBean schedulerFactoryBean;

    .....

    public Date loadJob(JobObject jobObject){
        //获取Scheduler实例
        Scheduler scheduler = schedulerFactoryBean.getScheduler();

        ....
        //指定Trigger和JobDetail进行注册    
        Date nextFireDate = scheduler.scheduleJob(jobDetail, trigger);

        return nextFireDate;
    }
}
//Spring配置文件中指定Bean
<bean id="schedulerFactory" class="org.springframework.scheduling.quartz.SchedulerFactoryBean" />

2. 整合Maven依赖

​ pom配置文件如下：

<dependency>
    <groupId>org.quartz-scheduler</groupId>
    <artifactId>quartz</artifactId>
    <version>2.2.1</version>
</dependency>

<dependency>
    <groupId>org.quartz-scheduler</groupId>
    <artifactId>quartz-jobs</artifactId>
    <version>2.2.1</version>
</dependency>
