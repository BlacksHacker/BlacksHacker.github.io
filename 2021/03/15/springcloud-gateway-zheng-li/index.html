SpringCloud： Gateway整理
SpringCloud Gateway
SpringCloud
发布日期:   2021-03-15
阅读次数:  

  • Spring Cloud 推出的二代网关框架，目的是取代Zuul

  • 功能如下：

    • 路由转发
    • 权限校验
    • 限流控制

  • 网关使用要配合服务注册，像eureka、nacos等

  • 所有服务均要注册到服务中心，包括网关服务同样需要注册。

    参考博客

    • https://blog.csdn.net/qq_38380025/article/details/102968559

核心依赖（引入到网关服务模块中）

       <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-starter-gateway</artifactId>
        </dependency>

转发配置的两种方式

  • 代码配置

    @SpringBootApplication
public class GatewayClientApplication {

    //uri 配置：test.uri: lb://service-one
    @Value("${test.uri}")
    private String uri;

    //代码配置
    @Bean
    public RouteLocator customRouteLocator(RouteLocatorBuilder builder) {
        return builder.routes()
                //basic proxy
                .route(r -> r.path("/orderApi/**")
                        .uri(uri)
                ).build();
    }

    public static void main(String[] args) {
        SpringApplication.run(GatewayClientApplication.class, args);
    }
}

    // 以上代码保证：http://localhost:8080/orderApi/order/info => http://localhost:8081/order/info（疑问：是否是http://localhost:8081/orderApi/order/info）

  • 配置文件配置

    # 服务在注册中心的名称  lb://开头 表示从注册中心获取服务
test:
uri: lb://service-one
spring:
application:
  name: gateway-client
cloud: # 网关配置
  gateway:
    routes:
    - id: route_service_one
      uri: ${test.uri} # uri以lb://开头（lb代表从注册中心获取服务），后面接的就是你需要转发到的服务名称
      predicates: # 通过网关访问时的请求前缀
      - Path=/userApi/**    # 保证了http://localhost:8080/userApi/user/who => http://localhost:8081/user/who (疑问：是否是http://localhost:8081/userApi/user/who) 
      filters: 
     - StripPrefix=1 # 表示在转发时去掉首层路径（因此会跳到http://localhost:8081/user/who 中）
server:
port: 8080
logging:
level:
  org.springframework.cloud.gateway: TRACE
  org.springframework.http.server.reactive: DEBUG
  org.springframework.web.reactive: DEBUG
  reactor.ipc.netty: DEBUG
eureka:
client:
  service-url:
    defaultZone: http://localhost:8761/eureka/
instance:
  prefer-ip-address: true

网关限流（令牌桶算法）

引入pom文件
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-gateway</artifactId>
</dependency>

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifatId>spring-boot-starter-data-redis-reactive</artifactId>
</dependency>
配置文件
server:
  port: 8080
spring:
  cloud:
    gateway:
      routes:
        - id: limit_route
          uri: http://httpbin.org:80/get
          predicates:
          - After=2019-02-26T00:00:00+08:00[Asia/Shanghai]
          filters: # 加入限流
          - name: RequestRateLimiter
            args:
              key-resolver: '#{@apiKeyResolver}' #用于限流的键的解析器的Bean对象的名字，通过SpEL表达式从容器中进行获取Bean对象
              redis-rate-limiter.replenishRate: 300 #设定每秒填充令牌速率
              redis-rate-limiter.burstCapacity: 500 #设定令牌的最大容量
              redis-rate-limiter.requestedTokens: 1 #设定单请求获取令牌的数量
  application:
    name: gateway-limiter
  redis:
    host: localhost
    port: 6379
    database: 0
编写限流解析器
@Configuration
public class RequestRateLimiterConfig {

    @Bean
    @Primary
    KeyResolver apiKeyResolver() {
        //按URL限流,即以每秒内请求数按URL分组统计，超出限流的url请求都将返回429状态
        return exchange -> Mono.just(exchange.getRequest().getPath().toString());
    }

    @Bean
    KeyResolver ipKeyResolver() {
        //按IP来限流(为了获取真实IP，而防止获取代理的IP)
        return exchange -> Mono.just(exchange.getRequest().getHeaders().getFirst("X-Forwarded-For"));
    }

    @Bean
    public KeyResolver hostAddrKeyResolver() {
        //按IP来限流（可能会限制代理IP）
        return exchange -> Mono.just(exchange.getRequest().getRemoteAddress().getHostName());
    }

}
