SpringCloud：OpenFeign与Hystrix整合
SpringCloud OpenFeign Hystrix
SpringCloud
发布日期:   2021-03-15
阅读次数:   
Spring Cloud 框架提供了RestTemplate和FeignClient两个方式完成服务间的调用，本文只要记录OpenFeign的使用，即FeignClient的形式调用。

1. 对应接口添加@FeignClient


// @AliasFor("value")
//    String name() default "";
//注解中的属性name和value的作用是一样的
//额外提醒，只要是传参，请都加上@RequestParam（"XXXX"）;如果传对象就加上@RequestBody； 这样你就不用踩Feign的传参的坑了，不管是使用Feign的服务还是提供接口的client服务，请都加上这些传参注解保持一致
@FeignClient(name = "captcha-service", fallbackFactory = CaptchaServiceFeignFallbackFactory.class)
public interface CaptchaServiceFeign {

    /**
     * 图片验证码
     * @return
     */
    @RequestMapping(value="/captcha/feign/imageCaptcha", method = RequestMethod.GET)
    CaptchaVo imageCaptcha();
}

2. 启动类添加注解@EnableFeignClients

//@EnableFeignClients(basePackages = "com.tcl.tcloud.base.common.feign")
//添加package，指明要使用FeignClient的位置
//不需要使用@RestTemplate,feign已经自动集成了Ribbon负载均衡的RestTemplate
@SpringBootApplication(exclude={DruidDataSourceAutoConfigure.class})
@EnableDiscoveryClient
@EnableFeignClients
@EnableRetry
@EnableConfigurationProperties
@EnableSwagger2
public class AuthApplication {

    public static void main(String[] args) {
        SpringApplication.run(AuthApplication.class, args);
    }

    @LoadBalanced
    @Bean
    RestTemplate restTemplate() {
        return new RestTemplate();
    }
}

//EnableFeignClients可以指定其配置参数 defaultConfiguration，该配置将会对所有的FeignClient生效，部分版本不行，需要单独配置

//关于 Feign Client 的配置还有个比较有趣的现象，尽管对于全局的 Feign Client 已经能够通过配置文件来进行默认属性的配置，但是这个 pull 并没有进入最新 GA 的 Dalston SR3版本。该 pull 在这里 `FeignClient Configuration Properties by khannedy · Pull Request #1942 · spring-cloud/spring-cloud-netflix · GitHub ，可以看到该 pull 被安排进了 Edgware 版本。当前项目使用还是用的 Dalston，所以只能用 configuraiotn 的形式来为每个 FeignClient 单独配置

//示例：@EnableFeignClients(defaultConfiguration = DefaultFeignConfiguration.class)

//feign的配置类
@Configuration
public class DefaultFeignConfiguration {

    @Bean
    public Retryer feignRetryer() {
        return new Retryer.Default(1000,3000,3);
    }

    //定义日志级别
     @Bean
    public Logger.Level feignLoggerLevel(){
        return Logger.Level.BASIC;
    }
}


Feign支持4种级别：

- NONE：不记录任何日志信息，这是默认值。

- BASIC：仅记录请求的方法，URL以及响应状态码和执行时间

- HEADERS：在BASIC的基础上，额外记录了请求和响应的头信息

- FULL：记录所有请求和响应的明细，包括头信息、请求体、元数据。

FeignClient的注解参数作用

package org.springframework.cloud.openfeign;

import java.lang.annotation.Documented;
import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;
import org.springframework.core.annotation.AliasFor;

@Target({ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface FeignClient {

    //指定FeignClient的名称，如果项目使用了Ribbon，name属性使用目标微服务的名称
    @AliasFor("name")
    String value() default "";

    /** @deprecated */
    @Deprecated
    String serviceId() default "";

    String contextId() default "";

    @AliasFor("value")
    String name() default "";

    String qualifier() default "";

    //url一般用于调试，可以手动指定@FeignClient的调用地址
    String url() default "";

    //当发生404时，该字段如果为true，会调用decoder进行解码，否则抛FeignException
    boolean decode404() default false;

    //Feign的配置类，可自定义Feign的Encoder、Decoder、Loglevel、Contract
    Class<?>[] configuration() default {};
    //定义容错的处理类，当远程调用失败或超时，会调用对应接口的容错逻辑，fallback指定的类必须实现@FeignClient标记的接口
    Class<?> fallback() default void.class;
    //用于生成fallback类示例，可实现每个接口通用的容错逻辑
    Class<?> fallbackFactory() default void.class;
    //定义当前FeignClient的统一前缀
    String path() default "";

    boolean primary() default true;
}

Feign实现Hystrix的两种方式

配置内容

feign:
  hystrix:
    enabled: true
    # 开启feign的熔断功能，默认false不会开启

方式一：

实现被@FeignClient注解标识的接口

public class CaptchaServiceFeignImpl implements CaptchaServiceFeign{
     CaptchaVo imageCaptcha(){
            //实现代码，即服务熔断之后要做的业务操作等等
         return null;
     }    
}

引用该类到@FeignClient的fallback属性上

@FeignClient(name = "captcha-service", fallback = CaptchaServiceFeignImpl.class)

方式二：

编写实现FallbackFactory接口的类

//将泛型参数 AppCenterClient 换成 CaptchaServiceFeign 即可

@Component
class ApplicationAuthFeignFallbackFactory implements FallbackFactory<AppCenterClient> {

    private final static Logger logger = LoggerFactory.getLogger(ApplicationAuthFeignFallbackFactory.class);

    @Override
    public AppCenterClient create(Throwable e) {
        return new AppCenterClient() {
            @Override
            public String checkApp(String appId, String appSect, String tenantId) {
                logger.error("verify appId and appSecret into fallback error:{}", e.getMessage());
                return null;
            }

            @Override
            public String getAppInfo(String appId, String tenantId) {
                logger.error("get app info into fallback error:{}", e.getMessage());
                return null;
            }

            @Override
            public String getAppInfos(String appId) {
                logger.error("get app infos into fallback error:{}", e.getMessage());
                return null;
            }
        };
    }
}

引用该类到@FeignClient的fallbackFactory属性上

@FeignClient(value = "base-app-service", fallbackFactory = ApplicationAuthFeignFallbackFactory.class)

        }

    //该类需要在@Configuration修饰的配置类中声明

public class StashErrorDecoder implements ErrorDecoder {

    @Override
    public Exception decode(String methodKey, Response response) {
        if (response.status() >= 400 && response.status() <= 499) {
            //这里是给出的自定义异常
            return new StashClientException(
                    response.status(),
                    response.reason()
            );
        }
        if (response.status() >= 500 && response.status() <= 599) {
            //这里是给出的自定义异常
            return new StashServerException(
                    response.status(),
                    response.reason()
            );
        }
        //这里是其他状态码处理方法
        return errorStatus(methodKey, response);
    }
}
  • Feign除了解决微服务之间的调用，里面还囊括了Ribbon负载均衡和Hystrix熔断降级

总结

  1. 首先，Feign内部已经包含了Ribbon和Hystrix,所以启动类加@EnableFeignClients就已经够用了，无需再加其他多余的相关Hystrix和Ribbon的注解

  2. Feign负责调用，其本质是对Ribbon通过RestTemplate进行请求调用的模板化封装

  3. Feign、Ribbon和Hystrix的关系是：

    • Feign负责发起调用
    • Ribbon负责负载均衡，其调用超过超时时间就切换被调用的实例
    • Hystrix负责熔断，接口调用时间（实例数 * ribbon的超时时间）大于熔断超时时间时将熔断，返回callback内容。

  4. Feign支持全局配置和单个Client配置

  5. Feign支持Configuration和YAML两种配置方式

  6. Feign可配置的内容包括

    • 日志配置

      • 使用loggin.level.来开启指定Feign客户端的DEBUG日志
      • 要使用完整路径
      • 需要开启日志级别，即Logger.Level，默认为NONE,推荐开启全局日志，通过bean或配置文件均可

    • 请求压缩

      • 开启请求和响应压缩

        feign.compression.request.enabled=true
feign.compression.response.enabled=true

      • 可以配置部分参数

        feign.compression.request.enabled=true
feign.compression.request.nime-types=text/xml,application/xml,application/json
feign.compression.requestmin-request-size=2048

  7. Feign超时配置

    #设置所有的微服务 超时配置
feign:
  client:
    config:
      default:
        connectTimeout: 50000
        readTimeout: 50000
        loggerLevel: basic
#单独设置某个微服务的超时配置
feign:
  client:
    config:
      server-A: # 设定server-A 服务调用的超时设置
        connectTimeout: 50000
        readTimeout: 50000
        loggerLevel: basic

 # 如果未配置feign的超时，配置ribbon也会生效，因为feign的底层是使用ribbon。

  1. Ribbon已经集成在了Feign中，可以直接在application中进行相关参数配置后使用

  2. Ribbon配置支持全局配置和指定服务配置

    • 全局配置：ribbon. =
    • 指定服务配置：.ribbon.key=value,client代表服务名称

  3. Ribbon负载均衡

    # 7种负载均衡中的一个（默认采用轮询）
ribbon.NFLoadBalancerRuleClassName=com.netflix.loadbalancer.ZoneAvoidanceRule

    实例化上述策略

    @EnableEurekaClient
@SpringBootApplication
@EnableFeignClients(basePackages = { "com.kyle.client.feign.inter" })
public class Application {

    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }

    //实例化策略
    @Bean
    public IRule feignRule() {
        return new ZoneAvoidanceRule();
    }
}

  4. Feign第一次请求失败

    • Spring采用懒加载机制i，熔断时间默认为1s，应该提升熔断超时时间和ribbon超时时间

  5. Feign的Http Client

    • 默认采用JDK原生URLConnection

    • 声明Apache HTTP Client和feign-httpclient依赖

    • application.properties添加

      feign.httpclient.enabled=true

  6. Feign支持请求之前进行操作，可对request进行修改，包括header和body。

    • 实现RequestInterceptor接口，实现apply方法

      @Component
public class TokenRequestInterceptor implements RequestInterceptor {
    @Override
    public void apply(RequestTemplate template) {
        String method = template.method();
        String url = template.url();
    }
}
