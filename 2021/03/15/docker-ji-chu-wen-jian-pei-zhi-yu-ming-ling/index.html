Dockerfile命令详解

  • FROM

  • 定制的镜像基于FROM的镜像

  • RUN

    • 用于执行后面跟着的命令行命令。

  • COPY

    • 从上下文目录复制文件或目录到容器指定路径

  • ADD

    • 和COPY的使用格式一致，比较推荐
    • 执行tar压缩文件，会自动复制解压到目标路径

  • CMD

    • 类似于RUN命令
      • CMD在docker run时执行
      • RUN在docker build时执行
      • 作用：为启动容器指定默认要启动的程序，可被docker run命令行参数中指定运行的程序覆盖。

  • ENTRYPOINT

    • 类似于CMD指令，但不会被run命令覆盖，会被当做参数传递给ENTRYPOINT指令指定的程序，当加了–entrypoint选项，其程序会被该参数指定的程序所覆盖。
    • 执行docker run 可以指定ENTRYPOIT

  • ENV

    • 设置环境变量

  • ARG

    • 构建参数，与ENV作用一致
    • 仅对Dockerfile内有效，即docker build的过程有效
    • 构建好的镜像内不存在此环境变量

  • VOLUME

    • 定义匿名数据卷，若启动忘记挂载，会自动挂载到匿名卷

  • EXPOSE

    • 仅仅是声明端口
    • 在运行时随机端口映射时，会自动随机映射EXPOSE的端口

  • WORKDIR

    • 指定工作目录

  • USER

    • 用于指定执行后续命令的用户和用户组

  • HEALTHCHECK

    • 用于指定某个程序或者指令来监控docker容器服务的运行状态

  • ONBUILD

    • 用于延迟构建命令的执行
    • 其命令内容本此镜像构建不会执行，当被作为引用镜像，参与构建时，该命令会被执行。

docker-compose.yml 内容详解

  • image
    • 镜像地址，仓库找不到时会尝试进行拉取
  • build
    • 指定Dockerfile所在文件夹路径，Compose会利用该文件自动构建这个镜像
    • build构建镜像和image下拉镜像，优先通过build执行Dockerfile构建镜像
  • command
    • 覆盖掉默认的命令
  • links
    • 可以link到其他的容器（同YAM文件中）
    • 可写应用名，或link别名
  • depends_on
    • 依赖的容器名
  • external_links
    • 可以link到非同YAML内的容器
    • 确保被link容器要正常运行
    • 可以直接写应用名，或别名
  • extra_hosts
    • hostname映射
    • 相当于Docker Run 中–add-host参数
  • ports
    • 申明主机和容器端口
    • 也可只声明容器端口，外部端口会随机选择
  • expose
    • 开放端口但不会被映射
    • 用于被其他的容器link
    • 只能保留内部端口
  • volumes
    • 支持Mount存储卷
    • 可指定主机路径和容器路径
    • 还可以包括只读
  • volumes_from
    • 支持从其他APP和容器Mount
  • environment
    • 添加环境变量
    • 可以指定YML数组或者字典
  • env_file
    • 从文件中获取文件变量
    • 通过docker-compose -f FILE 指定模板文件，则路径会基于模板文件路径
    • 变量名与environment指令冲突，则以后者为准
  • extends
    • 基于已有的服务进行扩展

docker 常见命令

Docker容器使用

  1. 指定容器执行命令

    docker run ubuntu:15.10 /bin/echo "Hellow world"

  1. 启动并进入容器终端

    docker run -i -t ubuntu:15.10 /bin/bash

docker run -it ubuntu:15.10 /bin/bash

  2. 容器后台模式（创建进程）

    docker run -d ubuntu:15.10 /bin/sh -c ""   ---d  带d会自动进入后台

docker run -itd --name ubuntu-test ubuntu /bin/bash

  3. ​ 宿主主机内查看容器标准输出

    docker logs 序列号

  4. 停止容器

    docker stop 序列号/名称

  5. 镜像获取

    docker pull ubuntu

  6. 查看所有容器

    docker ps -a

  7. 启动容器

    docker restart 序列号

docker start 序列号

  8. 进入后台容器

    docker attach 序列号                        ---------退出会导致容器停止

docker exec -it 序列号 /bin/bash             --------退出不会导致容器停止

  9. 导出容器

    docker export 序列号 > ubuntu.tar     导出镜像到指定位置

  1. 导入容器快照

    cat ubuntu.tar | docker import -test/ubuntu:v1   导入到镜像V1中

docker import http://example.com/exampleimage.tgz example/imagerepo   通过URL或目录导入

  2. 删除容器

    docker rm -f 序列号

  3. 查看docker映射端口

    docker port 序列号

  4. 查看容器内部进程

    docker top 实例名

  5. 检查docker底层信息 

    docker inspect 实例名
Docker镜像使用

  1. 查看本机镜像

    docker images

  2. 使用镜像运行容器

    docker run -it ubuntu:15.10 /bin/bash

  3. 获取新镜像

    docker pull ubuntu:13.10

  4. 查找镜像

    docker search httpd   ---查找httpd镜像
    Docker绑定端口和IP
    docker run -d -p 127.0.0.1:5000:5000/udp training/webapp python app.py   ---/udp  申明绑定UDP端口，默认绑定TCP接口
    Docker容器互联
    创建网络
docker network create -d bridge test-net

连接容器test1
docker run -itd --name test1 --network test-net ubuntu /bin/bash

连接容器test2
docker run -itd --name test2 --network test-net ubuntu /bin/bash

ubuntu 安装ping
apt-get update
apt install inputils-ping

ping 测试
ping test2

Dockerfile和docker-compose的区别

  • Dockerfile：用于构建镜像，包含镜像运行所需要的环境、程序代码等。
  • docker-compose：用于启动服务，编排容器，为容器设定启动参数、环境变量、容器命名，指定不同容器的链接参数等等。
