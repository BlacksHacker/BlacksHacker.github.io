ElasticSearch：基础点整理

ElasticSearch

1. ElasticSearch简介

  • 基于Lucene的高扩展的分布式搜索服务器，支持开箱即用。
  • 隐藏了Lucene的复杂性，对外提供Restful接口来操作索引、搜索。
  • 优点：
    • 扩展性好，可部署上百台服务器集群，处理PB级数据
    • 近实时的去索引数据、搜索数据

1.1 基本原理

​

  • 上面逻辑部分为倒排索引表，将文档内容进行分词，所有不重复的词组成分词列表。
  • 将搜索的文档最终以Document方式存储起来，每个词都和document关联。
  • 搜索时，将根据词与文档的关系，对文档进行检索输出。

1.2 零碎概念

  • 集群分扩容与收缩就是分片在节点上的迁移。
  • 分片分为主分片（primary shard）和复制分片(replica shard),文档都属于单独的主分片，其数量决定索引存储文件的数量。
  • 复制分片为主分片副本，提供读请求，比如搜索，或从别的shard取回文档
  • 索引创建完毕，主分片数量固定，复制分片可随时调整
  • 默认索引会被分配到5个主分片
  • 文档的索引首先存储于主分片中，然后并发复制到对应的复制节点上。
  • 启动新的节点，确保cluster.name相同的情况下，会自动加入集群，并进行分片迁移。
  • 节点固定，增加复制分片，性能不会提高，因为物理资源有限，但更多冗余可防止节点故障
  • PUT和POST的区别是：POST可以创建和更新，创建可以不指定ID，PUT可以创建和更新，创建必须指定ID
  • 删除不会立即删除，只是被标记为已删除，系统会在后续操作中进行删除，删除也会进行版本更新。
  • ES只会存储最后被索引的任何文档，即只存储最后一次的修改内容

2.ES操作语法

2.1 常用操作

  • 创建索引，分配主分片和复制分片（每个主分片都有一个复制分片）

    PUT /blogs
{
    "settings" : {
        "number_of_shards" : 3,
        "number_of_replicas" : 1
    }
}

  • 动态变更复制分片数量

    PUT /blogs/_settings
{
  "number_of_replicas": 2
}

  • 指定ID生成文档(行数据)

    PUT /website/blog/123
{
  "title" : "my first blog",
  "text" : "Just try this out...",
  "date" : "2020-12-24"
}

  • 自动生成ID

    POST /website/blog/
{
  "title" : "my first blog",
  "text" : "Just try this out...",
  "date" : "2020-12-24"
}

  • 检索指定文档的指定字段

    GET /website/blog/123?_source=title,date

  • 只要数据，不要元数据

    GET /website/blog/123?_source

  • 检查文档存在性(查询这一刻存在)

    HEAD /website/blog/123

  • 更新文档会标记移除旧的文档，索引到新的文档

    POST /website/blog/123
{
  "title" : "my third blog",
  "text" : "Just try this out...",
  "date" : "2020-12-24"
}

  • 确保文档不存在时才进行创建, op_type=create 或者_create

    PUT /website/blog/789/_create
{
  "title" : "my third blog",
  "text" : "Just try this out...",
  "date" : "2020-12-24"
}

  • 删除也会导致版本号增加

    DELETE /website/blog/XjrKk3YBXe9T4Gbt0pt0

  • 删除和更新可以指定if_seq和if_primary_term（允许代码中添加乐观锁）

    DELETE /website/blog/789?if_seq_no=10&if_primary_term=1

  • 局部局部更新（在检索和重建索引中保持更小的窗口）

    • 使用doc进行参数扩展

      POST /website/blog/123/_update
{
  "doc" : {
    "tags":["testing"],
    "view": 0
  }
}

    • 对字段进行累加

      POST /website/blog/123/_update
{
  "script":"ctx._source.view+=1"
}

    • 更新不存在的文档时，upsert中的参数会被用来进行新文件的创建

      POST /website/blog/111/_update
{
  "script":"ctx._source.views+=1",
  "upsert":{
    "views":1
  }
}

    • 设置冲突重试次数为5

      POST /website/blog/111/_update?retry_on_conflict=5
{
  "script":"ctx._source.views+=1",
  "upsert":{
    "views":1
  }
}

  • 多文档检索

    • 通过_mget命令，指定各文档的三个元数据进行检索，检索若在同一索引或其他条件下，可以设置统一默认值
    GET /_mget
{
  "docs":[
    {
      "_index":"website",
      "_type":"blog",
      "_id":"789"
    },
    {
      "_index":"website",
      "_type":"blog",
      "_id":"123",
      "_source":"views"  //检索确定的字段的值
    }
    ]
}

GET /website/blog/_mget
{
“docs”:[
{“_id”: 111},
{“_type” : “blog”, “_id”: 789}
]
}

//同索引，同类型下的不同文件检索
GET /website/blog/_mget
{
“ids”:[“111”, “789”]
}




#### 2.2 DSL查询（Query DSL-Domain Specific Language特定领域语言）

- **match**
  - 用法：使用match时，分词器会将其进行分词，凡是about中含有 love to go 任意一个单词 都会被检索到。

````json
GET /megacorp/employee/_search
{
  "query": {
    "match": {
      "about": "love to go"
    }
  }
}

  • match_phrase

    • 用法：短语搜索，要求所有分词必须同时出现在文档中，同时位置必须相邻，且顺序保持一致。
    GET /megacorp/employee/_search
{
  "query": {
    "match_phrase": {
      "about": "rock climbing"
    }
  }
}

  • term

    • 用法：精确查找，完全匹配，不会进行分词拆解，只能查单个词
GET /megacorp/employee/_search
{
  "query": {
    "term": {
      "about" : "like"
    }
  }
}

# like to 会检索不到结果
  • terms
    • 用法：匹配多词查询，只要满足其中一个就可以查出来（个人感觉和match 功能基本一样，写法有区别）
GET /megacorp/employee/_search
{
  "query": {
    "terms": {
      "about": [
        "love",
        "to"
      ]
    }
  }
}
  • range
    • 用法：允许指定范围查找一批数据
GET /megacorp/employee/_search
{
  "query": {
    "range": {
      "age": {
        "gte": 35,
        "lte": 40
      }
    }
  }
}

# gt:大于 gte: 大于等于 lt: 小于 lte:小于等于
  • exist
    • 用法：检索包含指定filed的一批数据
GET /megacorp/employee/_search
{
  "query": {
    "term": {
      "about" : "love to"
    }
  }
}
  • bool
    • 用法：布尔过滤器，复合过滤器，可接受多个其他过滤器作为参数
{
    "bool" : {
        "must" : [],    //等价于 AND
        "should" : [],  //等价于 OR
        "must_not" : [],  //等价于 NOT
        "filter" : {},
        "boost" : {}
    }
}
// 查询结果中至少要有 chrome 或 firfox (个人感觉很像 terms)
{
    "bool" : {
        "should" : [   // 换成 must,代表必须匹配 AND 关系，换成must_not 表示必须不能含有
            {
                "term" : {
                    "title" : "chrome"
                }
            },
            {
                "term" : {
                    "title" : "firfox"
                }
            }
        ]
    }
}
{
    "bool" : {
        "should" : [
            {
                "term" : {
                    "title" : "chrome"
                }
            },
            {
                "term" : {
                    "title" : "firfox"
                }
            },
            {
                "term" : {
                    "title" : "safari"
                }
            }
        ],
        "minimun_should_match" : 2 //匹配的结果中，至少要包含其中的两个 
    }
}
SELECT document
FROM   products
WHERE  productID      = "KDKE-B-9947-#kL5"
  OR (     productID = "JODL-X-1937-#pV7"
       AND price     = 30 )

上面的sql转化为ES查询为：

{
    "query" : {
        "bool" : {
            "should" : [
                {"term" : {"productID" : "KDKE-B-9947-#kL5"}},
                {"bool" : {
                    "must" : [
                        {"term" : {"productID" : "JODL-X-1937-#pV7"}},
                        {"term" : {"price" : "30"}}
                    ]
                }}
            ]
        }
    }    
}

# 逐渐从外层由大范围进行分解，即最外层先是or，然后是and
  • match_all
    • 用法：检索所有数据
GET /megacorp/employee/_count  //统计所有文件的数量
{
  "query":{
    "match_all":{}
  }
}
